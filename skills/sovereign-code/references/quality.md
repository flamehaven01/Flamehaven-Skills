# Sovereign Code Quality Dimensions (5D)

## Quality Formula

**Q = 0.30×Correctness + 0.25×Readability + 0.20×Performance + 0.15×Security + 0.10×Maintainability**

## Dimensions

1. **Correctness (30%)**
   - Logic correctness
   - Type safety
   - Edge case handling
   - Error handling
   - Input validation

2. **Readability (25%)**
   - Code clarity
   - Naming conventions
   - Code structure
   - Documentation
   - Comments quality

3. **Performance (20%)**
   - Time complexity
   - Space complexity
   - Algorithm efficiency
   - Resource optimization
   - Scalability

4. **Security (15%)**
   - Vulnerability-free code
   - OWASP compliance
   - Input validation
   - Authentication/Authorization
   - Data encryption

5. **Maintainability (10%)**
   - Modular design
   - Testability
   - Extensibility
   - Code reusability
   - Dependency management

## Grading Scale

- **S-Grade (≥0.95)**: Exceptional - Production excellence
- **A-Grade (≥0.90)**: Excellent - Production ready
- **B-Grade (≥0.80)**: Good - Minor improvements needed
- **C-Grade (≥0.70)**: Acceptable - Moderate improvements needed
- **D-Grade (≥0.60)**: Poor - Major improvements required
- **F-Grade (<0.60)**: Failing - Complete rewrite recommended

## When to Use

- User requests code generation ("함수 만들어줘")
- API or service development
- Algorithm implementation
- Code scaffolding
- Feature implementation
